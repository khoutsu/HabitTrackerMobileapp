# เอกสารข้อกำหนดผลิตภัณฑ์ (PRD) - Loop Habit Tracker

## 1. บทนำ

**ชื่อผลิตภัณฑ์:** Loop Habit Tracker

เอกสารนี้สรุปข้อกำหนดของผลิตภัณฑ์สำหรับแอปพลิเคชันมือถือ Loop Habit Tracker แอปนี้ออกแบบมาเพื่อช่วยให้ผู้ใช้สร้าง ติดตาม และรักษานิสัยที่ดี โดยมีอินเทอร์เฟซที่เรียบง่ายแต่ทรงพลังเพื่อให้ผู้ใช้ติดตามความคืบหน้า สร้างแรงจูงใจ และสร้างกิจวัตรที่ยั่งยืน

**กลุ่มเป้าหมาย:** บุคคลที่กำลังมองหาเครื่องมือในการปรับปรุงวินัยในตนเอง สร้างทักษะใหม่ๆ หรือเพียงแค่จัดระเบียบกิจวัตรประจำวัน

## 2. เป้าหมายและวัตถุประสงค์

*   **เป้าหมายหลัก:** เพื่อให้ผู้ใช้มีเครื่องมือที่ครอบคลุมและใช้งานง่ายในการสร้างและรักษานิสัย
*   **วัตถุประสงค์หลัก:**
    *   สร้างและปรับแต่งนิสัยได้อย่างง่ายดาย
    *   ให้ข้อเสนอแนะที่ชัดเจนและลึกซึ้งเกี่ยวกับความคืบหน้าของผู้ใช้ผ่านสถิติและภาพข้อมูล
    *   สร้างแรงจูงใจให้ผู้ใช้ด้วยฟีเจอร์ต่างๆ เช่น สถิติการทำต่อเนื่อง (streaks) และคะแนนนิสัย
    *   รับประกันว่าข้อมูลผู้ใช้ปลอดภัยและสามารถสำรองและกู้คืนได้
    *   มอบประสบการณ์ผู้ใช้ที่สะอาดตา ปรับแต่งได้ และเข้าถึงง่าย

## 3. คุณสมบัติหลัก

### 3.1. การจัดการนิสัย

*   **สร้าง/แก้ไขนิสัย:** ผู้ใช้สามารถสร้างนิสัยใหม่พร้อมคุณสมบัติดังต่อไปนี้:
    *   ชื่อ (เช่น "อ่านหนังสือ", "ออกกำลังกาย")
    *   คำอธิบาย/คำถาม (เช่น "คุณอ่านหนังสืออะไร")
    *   หมวดหมู่ (เช่น "สุขภาพ", "การเรียนรู้")
    *   ความถี่ (เช่น รายวัน, วันที่ระบุในสัปดาห์, จำนวนครั้งต่อสัปดาห์)
*   **จัดเก็บนิสัย (Archive):** ผู้ใช้สามารถจัดเก็บนิสัยที่ไม่ต้องการติดตามอีกต่อไป เพื่อนำออกจากรายการหลัก แต่ยังคงประวัติไว้
*   **ดูรายการนิสัย:** หน้าจอหลักแสดงรายการนิสัยที่ใช้งานทั้งหมดสำหรับวันปัจจุบัน

### 3.2. การติดตามนิสัย

*   **ทำเครื่องหมาย:** ผู้ใช้สามารถทำเครื่องหมายนิสัยว่า "ทำแล้ว", "ข้าม" หรือ "ไม่ได้ทำ" ในแต่ละวัน
*   **มุมมองรายละเอียดนิสัย:** มุมมองโดยละเอียดสำหรับแต่ละนิสัยจะแสดง:
    *   มุมมองปฏิทินของประวัติการทำนิสัย
    *   สถิติการทำต่อเนื่องปัจจุบันและดีที่สุด (streak)
    *   อัตราความสำเร็จ (เปอร์เซ็นต์ของวันที่ทำสำเร็จ)
    *   คะแนนนิสัย
    *   แผนภูมิความถี่

### 3.3. การแจ้งเตือน

*   **การแจ้งเตือนที่กำหนดเอง:** ผู้ใช้สามารถตั้งค่าการแจ้งเตือนหลายรายการสำหรับแต่ละนิสัยเพื่อรับการแจ้งเตือนในเวลาที่กำหนด

### 3.4. สถิติและการวิเคราะห์

*   **สถิติการทำต่อเนื่อง (Streaks):** แอปจะคำนวณจำนวนวันที่ทำนิสัยติดต่อกัน
*   **อัตราความสำเร็จ:** แอปจะคำนวณเปอร์เซ็นต์ของจำนวนครั้งที่ทำนิสัยสำเร็จตลอดอายุการใช้งาน
*   **คะแนนนิสัย:** ระบบการให้คะแนนเพื่อวัดความแข็งแกร่งของนิสัย

### 3.5. การจัดการข้อมูล

*   **สำรองและกู้คืน:** ผู้ใช้สามารถส่งออกข้อมูลแอป (นิสัย, ประวัติ) เป็นไฟล์เพื่อสำรองข้อมูลและนำเข้าในภายหลังเพื่อกู้คืนข้อมูล ซึ่งช่วยป้องกันข้อมูลสูญหาย

### 3.6. ประสบการณ์ผู้ใช้และการปรับแต่ง

*   **การเริ่มต้นใช้งาน (Onboarding):** คำแนะนำสำหรับผู้ใช้ครั้งแรกเพื่ออธิบายคุณสมบัติหลักของแอป
*   **การปรับแต่งธีม:** ผู้ใช้สามารถสลับระหว่างธีมสว่างและธีมมืดของแอปพลิเคชันได้
*   **การแปล (Localization):** แอปรองรับหลายภาษา รวมถึงภาษาอังกฤษและภาษาไทย
*   **วิดเจ็ตบนหน้าจอหลัก:** (อนุมานจาก `widget_service.dart`) ผู้ใช้สามารถเพิ่มวิดเจ็ตลงในหน้าจอหลักของอุปกรณ์เพื่อดูและอัปเดตนิสัยโดยไม่ต้องเปิดแอป

## 4. ข้อกำหนดที่ไม่ใช่ฟังก์ชันการทำงาน (Non-Functional Requirements)

*   **ประสิทธิภาพ:** แอปต้องรวดเร็วและตอบสนองได้ดี โดยมีเวลาในการโหลดน้อยที่สุด
*   **แพลตฟอร์ม:** พัฒนาด้วย Flutter สำหรับ Android และ iOS
*   **การจัดเก็บข้อมูล:** ข้อมูลผู้ใช้ทั้งหมดจะถูกเก็บไว้ในเครื่องบนอุปกรณ์ในฐานข้อมูล SQLite
*   **การเข้าถึง (Accessibility):** UI ควรเป็นไปตามแนวทางปฏิบัติที่ดีที่สุดในการเข้าถึง (เช่น คอนทราสต์ของสีที่เพียงพอ, แบบอักษรที่ปรับขนาดได้)

## 5. การพิจารณาในอนาคต (นอกขอบเขตสำหรับเวอร์ชันเริ่มต้น)

*   การซิงโครไนซ์ข้อมูลบนคลาวด์ข้ามอุปกรณ์หลายเครื่อง
*   การวิเคราะห์ทางสถิติและแผนภูมิขั้นสูง
*   คุณสมบัติโซเชียลสำหรับการแบ่งปันความคืบหน้ากับเพื่อน
*   เทมเพลตและคำแนะนำนิสัยที่กำหนดไว้ล่วงหน้า
*   การรวมเข้ากับอุปกรณ์สวมใส่ได้ (เช่น Apple Watch, Wear OS)
