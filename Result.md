# รายงานสรุปผลการดำเนินงานโปรเจกต์

## ขั้นตอนการดำเนินงาน

1. ศึกษาโครงสร้างของแอปพลิเคชันและวางแผนระบบการทำงาน
2. จัดการแก้ไขปัญหาในส่วนแสดงผล (UI/UX) ให้สามารถแสดงผลได้อย่างถูกต้อง
3. พัฒนาการเชื่อมต่อระบบแจ้งเตือน (Firebase Cloud Messaging) และตั้งค่าภาษา (Localization)
4. ทดสอบแอปพลิเคชัน ค้นหาข้อบกพร่อง และทำการแก้ไขให้พร้อมใช้งาน

## ปัญหาที่พบ

1. ปัญหาการแสดงผลของตัวอักษรล้นกรอบหน้าจอ (Text Overflow)
2. โครงสร้างของโปรเจกต์มีความซับซ้อน ทำให้หาตำแหน่งไฟล์เพื่อทำความเข้าใจได้ยาก
3. ปัญหาเกี่ยวกับการจัดการภาษาของระบบ ที่ทำให้เกิดข้อผิดพลาดคั่นการทำงาน
4. ปัญหาขั้นตอนการคอมไพล์ (Build Release APK) เพื่อนำแอปพลิเคชันไปรันใช้งานจริง

## แนวทางแก้ไข

1. ปรับปรุง Layout โครงสร้างหน้าจอให้มีความยืดหยุ่นและรองรับข้อความยาว
2. ใช้เครื่องมือช่วยค้นหาและไล่โครงสร้างโค้ดทีละส่วนเพื่อทำความเข้าใจ Flow ของระบบ
3. อัปเดตไฟล์รวบรวมคำแปลภาษา (.arb) และรันคำสั่งเพื่อรองรับคำศัพท์ใหม่
4. ตรวจสอบ Log ค้นหาสาเหตุ พร้อมจัดการแก้สิทธิ์การเข้าถึงอุปกรณ์ให้ถูกต้องและรองรับไลบรารี

## ปัญหาและอุปสรรคที่ทำให้การดำเนินงานล่าช้า

ปัญหาต่างๆ ที่พบระหว่างการพัฒนาส่งผลให้การดำเนินงานเกิดความล่าช้ากว่าที่วางแผนไว้ โดยมีรายละเอียดดังนี้:

1. **การทำความเข้าใจโครงสร้างโปรเจกต์เดิมที่มีความซับซ้อน:**
   ทำให้ในช่วงเริ่มต้นต้องใช้ระยะเวลาพอสมควรในการทำความเข้าใจสถาปัตยกรรมของแอปพลิเคชัน การไหลเวียนของข้อมูล (Data Flow) และไล่หาตำแหน่งไฟล์ที่เกี่ยวข้องที่ถูกต้องก่อนที่จะสามารถทำความเข้าใจและเริ่มเขียนโค้ดแก้ไขได้

2. **ข้อบกพร่องในสภาวะแวดล้อม Release (Release Build Issues):**
   พบปัญหาแอปพลิเคชันสามารถทำงานได้ปกติในโหมด Debug แต่ขัดข้องเมื่อคอมไพล์เป็น Release APK การตรวจสอบวิเคราะห์หาสาเหตุในโหมดนี้ทำได้ยากและใช้เวลานาน เนื่องจากต้องตรวบสอบการจัดการไลบรารี การตั้งค่า Build และสิทธิ์ (Permissions) ของระบบปฏิบัติการอย่างละเอียด

3. **ปัญหาความยุ่งยากของการจัดการตั้งค่าภาษา (Localization Errors):**
   การทำงานกับไฟล์แปลภาษา (`.arb`) มักเกิดปัญหาที่ตัวแปร (Getter) ชนกันหรือไม่บันทึกข้อมูลแบบเรียลไทม์ ส่งผลให้ขั้นตอนการสร้างไฟล์ภาษาอัตโนมัติ (Code Generation) หยุดชะงัก ซึ่งต้องอาศัยการไล่หาจุดที่ผิดพลาดอย่างระมัดระวังและใช้เวลาในการคอมไพล์ใหม่

4. **การรันและทดสอบความยืดหยุ่นของ UI และระบบแจ้งเตือน:**
   การปรับแก้ปัญหาพื้นที่แสดงผลข้อความล้นหลุดกรอบ (Text Overflow) รวมถึงปัญหาของระบบแจ้งเตือน จำเป็นต้องรันเพื่อทดสอบบนอุปกรณ์ที่มีขนาดหน้าจอแตกต่างกัน ทำให้ต้องใช้เวลาค่อนข้างมากไปกับการปรับแก้หน้าต่างและ Layout ให้ยืดหยุ่นเพื่อครอบคลุมข้อความภาษาต่างๆ ที่มีความยาวไม่เท่ากัน

## สรุปผลการดำเนินงาน

1. สามารถพัฒนาแอปพลิเคชันจำลองเป้าหมาย (Loop Habit Tracker) จนสามารถนำไปรันใช้งานได้จริง
2. ตัวแอปพลิเคชันมีความเสถียร แสดงผลการทำงานและแจ้งเตือนได้ตรงกับความต้องการของผู้ใช้งาน
