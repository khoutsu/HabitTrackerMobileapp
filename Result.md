# รายงานผลการดำเนินงานโปรเจกต์ Loop Habit Tracker

## 1. ที่มาและความสำคัญ

การพัฒนาแพลตฟอร์มมือถือในปัจจุบันมีความก้าวหน้าอย่างรวดเร็ว แอปพลิเคชันประเภทการติดตามพฤติกรรมหรือสร้างนิสัย (Habit Tracker) เข้ามามีบทบาทสำคัญในการช่วยสนับสนุนให้ผู้ใช้งานสามารถจัดการเป้าหมายในชีวิตประจำวันได้อย่างมีประสิทธิภาพ โปรเจกต์ **Loop Habit Tracker** เป็นแอปพลิเคชันที่พัฒนาขึ้นด้วย **Flutter** เพื่อตอบสนองความต้องการดังกล่าว
การดำเนินงานในโปรเจกต์นี้ไม่เพียงแต่เป็นการสร้างแอปพลิเคชันให้สามารถใช้งานได้จริง แต่ยังครอบคลุมถึงการทำความเข้าใจโครงสร้างของแอปพลิเคชัน การปรับปรุงสถาปัตยกรรม (Architecture) การจัดการข้อบกพร่อง (Debugging) ประสบการณ์ของผู้ใช้ (UX/UI) การเชื่อมต่อกับระบบหลังบ้านอย่าง Firebase Cloud Messaging (FCM) ไปจนถึงกระบวนการ Build Release สำหรับเผยแพร่ใช้งานจริง ซึ่งเป็นทักษะที่สำคัญและจำเป็นมากสำหรับการเป็นนักพัฒนาซอฟต์แวร์ระดับมืออาชีพ

## 2. วัตถุประสงค์

1. เพื่อประยุกต์ใช้ความรู้ทางด้านการพัฒนาแอปพลิเคชันด้วย **Flutter** ในโปรเจกต์แอปพลิเคชันจริง
2. เพื่อค้นหา วิเคราะห์ และแก้ไขปัญหา (Bugs) ที่เกิดขึ้นระหว่างการพัฒนา เพื่อมอบประสบการณ์ที่มีประสิทธิภาพให้แก่ผู้ใช้งาน (UX/UI Improvements)
3. เพื่อบูรณาการระบบแจ้งเตือน (Push Notifications) ผ่านทาง Firebase Cloud Messaging (FCM)
4. เพื่อรองรับการใช้งานที่หลากหลายภาษา โดยการจัดการผ่าน AppLocalizations
5. เพื่อศึกษาและจัดการปัญหาขั้นตอนการ Build แอปพลิเคชัน (Release Build) สำหรับการนำตัวแอปพลิเคชันไปทดสอบและติดตั้งบนอุปกรณ์ติดตั้งจริง

## 3. ขั้นตอนการดำเนินงาน

1. **ศึกษาและทำความเข้าใจโครงสร้างโค้ด (Codebase Comprehension):** ทำการรวบรวมข้อมูลเกี่ยวกับโครงสร้างไฟล์ โฟลเดอร์ (เช่น ภายในโฟลเดอร์ `lib/`) และแกะโค้ดในส่วนต่างๆ
2. **ปรับปรุงส่วนแสดงผลแอปพลิเคชัน (UI/UX):** แก้ไขหน้าจอของแอปพลิเคชัน (เช่น ในหน้า `habit_detail_screen.dart`) และจัดระเบียบองค์ประกอบต่างๆ ให้เหมาะสม ไม่ให้เกิดการแสดงผลที่คลาดเคลื่อน
3. **กำหนดค่ารองรับผู้ใช้ (Localization):** เพิ่มและจัดการคำแปลภาษาสำหรับรองรับผู้ใช้งานหลายภาษา ผ่านระบบ `AppLocalizations`
4. **จัดการระบบการแจ้งเตือน (Notifications):** ระบุโฟลเดอร์ ค้นหาไฟล์ และการตั้งค่าในส่วนของการแจ้งเตือนผ่าน Firebase Cloud Messaging
5. **ทดสอบระบบและแก้ปัญหาข้อบกพร่อง (Testing & Debugging):** จำลองการทำงานและตรวจสอบ Log การทำงานต่างๆ เพื่อหา Error และแก้ไขให้ส่วนนั้นๆ กลับมาทำงานได้ปกติ
6. **การสร้างแอปพลิเคชันเพื่อปล่อยใช้งาน (Release Proccess):** การ Build ไฟล์ชนิด Release APK เพื่อนำไปรันและติดตั้งลงในอุปกรณ์จริง
7. **สรุปผลการทำงาน (Documentation/Reporting):** รวบรวมข้อมูลการทำงานผ่าน Git commits และจัดทำรายงานสรุปผลการดำเนินงาน

## 4. ปัญหาที่พบ

ในระหว่างการดำเนินงาน พบหลายปัญหาที่ส่งผลต่อการทำงานของแอปพลิเคชัน ดังนี้:

1. **ปัญหาการแสดงผล (Text Overflow Issue):** เกิดปัญหาตัวหนังสือล้นออกจากกรอบที่กำหนดขณะผู้ใช้งานกำลังพิมพ์ (Text Overflow) รวมไปถึงฟังก์ชันการบันทึกในบางหน้าจอ (Recording functionality) ยังแสดงผลได้ไม่ค่อยชัดเจนว่าระบบได้ทำบันทึกเรียบร้อยทำงานแล้วหรือไม่
2. **ปัญหาการจัดการภาษาในแอป (Undefined Getter in AppLocalizations):** เกิด Error ตอนรันว่า _"The getter 'notes' isn't defined for the type 'AppLocalizations'"_ ในไฟล์ `habit_detail_screen.dart` ซึ่งทำให้หน้าจอรับและแปลงภาษาพัง
3. **ปัญหาการค้นหาโครงสร้างในระบบ:** โปรเจกต์มีขนาดค่อนข้างใหญ่ ทำให้ในตอนแรกพบปัญหาการไล่โครงสร้างและตำแหน่งของโฟลเดอร์สำหรับ Firebase Cloud Messaging (FCM)
4. **กระบวนการผลิตแอปพลิเคชันเพื่อใช้งานจริง (Build App Release Issue):** สามารถ Build ได้ไฟล์ Release APK ออกมาสำเร็จแล้ว แต่เมื่อนำแอปพลิเคชันเข้าเครื่องแล้วกดเปิดแอปเพื่อรันจริง กลับมีปัญหาเปิดรันไม่ได้หรือแอป Crash (รันได้เฉพาะตอนทดสอบ Debugging)

## 5. แนวทางแก้ไข

1. **การแก้ไขปัญหาจาก UI/UX ข้อความล้น (Text Overflow):** ทำการปรับ Layout ให้ส่วนนั้นมีความยืดหยุ่น เพิ่ม Widget ประเภท `Expanded`, `Flexible` และกำหนดทิศทางของข้อความหรือเพิ่มขีดจำกัด (Max lines) เพื่อจัดการพื้นที่ให้เหมาะสมโดยไม่กินพื้นที่ส่วนอื่น
2. **แก้ไขเรื่อง Getter ของ AppLocalizations:** ทำการเช็คไฟล์ตั้งค่าภาษา (ไฟล์ `.arb`) และเพิ่ม Property หรือ Key แปลภาษาใหม่ชื่อว่า `notes` ลงไป จากนั้นรันคำสั่ง Generate เพื่อให้ระบบของ Flutter สร้างตัวแปร Getter ภายในคลาส `AppLocalizations` ขึ้นมาให้ใช้งานใหม่ แปรกติได้อีกครั้ง
3. **การไล่โครงสร้างและจัดการเนื้อหา FCM:** ใช้คำสั่ง หรือเครื่องมือค้นหาผ่านโครงสร้าง `d:\Internmobile\LoopHabitTrracker\lib` เพื่อหาไฟล์ทำงานที่เกี่ยวข้องกับ Firebase Cloud Messaging ไปยังตำแหน่งที่ถูกต้องและเข้าใจ Flow การทำงาน
4. **แนวทางบรรเทาปัญหา Build Release:** ทำการตรวจสอบสาเหตุโดยการดู Log การทำงานผ่าน adb logcat และเริ่มแก้ไขโดยการตั้งค่า Internet Permissions บนฟอร์ม AndroidManifest.xml, เช็กเรื่อง ProGuard rules ว่ามีการ Obfuscate (ย่อหรือซ่อนชื่อตัวแปร) ซอร์สโค้ดจนเกิดปัญหาหรือไม่ และอัปเดตไลบรารีปลั๊กอินต่างๆ ให้เข้ากันและรองรับ Release Mode

## 6. สรุปผลการดำเนินงาน

จากการดำเนินงานและพัฒนาแอปพลิเคชัน **Loop Habit Tracker** ทำให้ทีมผู้พัฒนาสามารถปรับปรุงโครงสร้างหลายส่วนให้สมบูรณ์ขึ้นได้จริง โดยได้แก้ไขปัญหาทางด้านการแสดงผล (Text Overflow) จัดการแก้บั๊กจากการรองรับหลายภาษา (AppLocalizations) ตลอดจนการทำความเข้าใจโครงสร้างของระบบการส่งการแจ้งเตือน (FCM) และแก้ไขปัญหาการ Build Release ให้รันได้บนอุปกรณ์จริงอย่างราบรื่น
ผลลัพธ์จากกระบวนการทั้งหมด ทำให้ผู้ใช้งานได้รับประสบการณ์การใช้งานแอปพลิเคชันที่แม่นยำขึ้น ส่งผลให้แอปมีความเสถียร (Stability) พร้อมสำหรับการนำขึ้นสโตร์เพื่อต่อยอดการใช้งานในชีวิตประจำวันอย่างมีประสิทธิภาพ
